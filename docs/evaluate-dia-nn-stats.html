What you’ll see is each run as it’s own row and then 16 different “measurements” of each run. The columns you are going to use most of the time are:

  • Precursors.Identified- The number of precursors identified in that run. It is important to note that a precursor is simply a charged peptide, so a single peptide sequence can have multiple precursors (i.e., if peptide x was both doubly and triply charges, that would count as 2 precursor IDs)

  • Proteins.Identified- The number of proteins identified in that run.

  • Average.Missed.Tryptic.Cleavages- The average number of missed cleavages by trypsin in that run. A lower number indicates a more efficient digestion.

  • Some typical numbers for various experiments might be:

    • Global experiment of mammalian cell lysate:

      • 90,000-140,000 Precursor IDs

      • 9,000-10,000 Protein IDs

      • 5-15% Missed Cleavages

    • Global experiment of E. coli lysate:

      • 35,000-45,000 Precursor IDs

      • 3,000-3,300 Protein IDs

      • 10-20% Missed Cleavages

    • Phospho experiment (varies greatly on type of material and amount enriched; assuming 100ug of cultured cells):

      • 30,000-40,000 Precursor IDs

      • 4,000-5,500 Protein IDs

      • 40-60% Missed Cleavages

    • Single-cell experiment:

      • 2,000-10,000 Precursor IDs (varies greatly depending on cell type and size)

      • 500-2,500 Protein IDs (varies greatly depending on cell type and size)

      • 10-20% Missed Cleavages

Since the stats report has a lot of information that isn’t necessarily important for a majority of users, let’s look at the columns we’re interesting in using dplyr::select().

Additionally, the file name unfortunately includes the entire file path of the raw file, making it harder to see and understand what each file is. We’re also going to pipe in a dplyr::mutate() to change the File.Name column to remove the common text in all the rows using the gsub function. This function looks for all strings that match the one entered in and then replaces those strings with one you tell it to (in this case it will be nothing or ""). And within this gsub(), you’ll also be seeing out first example of a regular expression or regex. Regex commands can be useful when working with lnong strings following a standardized “formula”. In this scenario, all files use the following string Y:\\9- Teaching Materials and Presentations\\Proteomics Summer Course\\2026\\Datasets\\E Coli Ahmed\\Raw Files\\Ahmed_ followed by the sample name. Instead of type out the first part, which is the same in every sample, we’re going to type Y.*Ahmed_, which simply means “Any string that starts with Y followed by any character (.) for an unlimited number of times (*), until we reach the string”Ahmed_“, at which point, we stop. And then that entire string will be replaced with "", which is nothing. ( We are also going to remove the ending string from each run since those are also the same for every one;”_25-023.raw” (the work order number used by the MSRL assigned to each project, followed by the file type of that run).

diann_stats %>% 
  # We only want some specific columns
  select(File.Name, # Name of the run
         Precursors.Identified, # The number of Precursor IDs
         Proteins.Identified, # The number of Protein IDs
         Average.Missed.Tryptic.Cleavages) %>% # The average number of missed cleavages in that run
  # We are going to the common sample monikers the MSRL uses. Specifically, each run starts with "<Researcher name>_" and ends with "_<Work order number>". For the dataw e will be using in this tutorial, the researcher name is "Ahmed" and the work order number is "25-023".
  # We are going to combine the 'mutate()' dplyr function with the 'sub()' base R function to change the contents of the "File.Name" column. Specifically, we are going to find all times either "Amhed_" or "_25-023" appear in the the column "File.Name" and replace those strings with "" (ie, nothing)
  mutate(File.Name = sub('Y.*Ahmed_', #
                         '',
                         File.Name),
         File.Name = sub('_25-023.raw',
                         '',
                         File.Name))

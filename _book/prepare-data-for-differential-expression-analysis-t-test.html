3.6 Prepare Data for Differential Expression Analysis (t-test)

There are just a few more things we need to do before our data is ready for statistical analysis. More specifically, we need to:

  1. Filter data based off total number of peptides and completeness

  2. Replace missing values through imputation

3.6.1 Filter Based off Number of Peptides and Missing Values

When dealing with biological data, especially when dealing with gene knockouts, it would not be unexpected for some proteins to not be detected at all in some groups compared to others. This could be a result of a combination of two things:

  1. The protein is either down-regulated or not even expressed in one group

  2. The level of the protein is beneath the limit of detection of our analytical method (in this case, the mass spectrometer)

The main issue that arises from this is that some proteins will be detected in some samples/groups and not others. The samples the protein was not detected in will have a value like 0 of NA denoting that there is no value present for that protein in that sample. This is called a missing value. R does not handle NA very well (or any programming language for that matter), so when we try to perform any kind of statistical analysis on groups where NA is present, it either will not work or will look “funky”. There are several ways to circumvent this program, including:

  • Removing any proteins with a missing value

  • Replacing missing values with a constant number

  • Imputing the missing value using some mathematical formula

The last method, imputation is usually how we go ahead with handling missing values. Specifically, a method where a random, low-value number is assigned to a missing value (at the protein level). More specifically:

  1. The protein abundances are converted to log2 (to make the data normally distributed)

  2. A standard curve is made from the log2 protein abundances

  3. That standard curve is shifted by a standard deviation we determine (we default to 1.8 standard deviations)

  4. The new, shifted curve is “shrunk” to a width of the original curve that we determine (we default to 0.3 the width of the original curve)

  5. A random number from this new, shifted and shrunken curve is assigned to each missing value

A key assumption to this method is that any missing values correspond to low-abundance proteins that fall below the limit of detection of our mass spectrometer.

However, before we impute, we perform one more quality check to make sure what we’re imputing isn’t artificially creating biological phenomena. Specifically, we set thresholds for each protein so that each protein:

  1. Has multiple peptides that map to that protein (no “one hit wonders”)

  2. There are multiple replicates within at least one group with multiple peptides

diann_peptide_counts <- diann_filter %>% 
  group_by(Run, Protein.Group, Genes) %>% 
  summarise(n_peptides = n(),
            .groups = 'drop') %>% 
  left_join(y = diann_filter %>% 
              distinct(Run, group, Protein.Group, Genes, PG.MaxLFQ),
            by = c('Run', 'Protein.Group', 'Genes'))

diann_peptide_filter_list <- diann_peptide_counts %>% 
  drop_na() %>% 
  select(-PG.MaxLFQ) %>% 
  filter(n_peptides >= 2) %>% 
  group_by(group, Protein.Group, Genes) %>% 
  summarise(n_reps = n(),
            .groups = 'drop') %>% 
  filter(n_reps >= 2) %>% 
  select(-n_reps)

diann_peptide_filtered_data <- inner_join(x = diann_peptide_counts,
                                          y = diann_peptide_filter_list,
                                          by = c('group', 'Protein.Group', 'Genes'))

We now have a data frame with a new column containing the number of peptides that were identified for each protein in each sample. And was filtered so that every protein was quantified using 2 or more peptides. AND has at least one group with at least two replicates with that protein present. So now we are a little more confident in our dataset that the proteins that are still present were at least quantified in at least one of our groups.

The last thing we need to do before imputing missing values is to make our data “wide”, which just means that instead of each run being on its own row, each row is now a single protein and each column is the protein abundance in that sample. At this point, we will now see NA if there is a missing value.

diann_peptide_filtered_wide <- diann_peptide_filtered_data %>% 
  select(-c(n_peptides, group)) %>% 
  mutate(PG.MaxLFQ = log2(PG.MaxLFQ)) %>% 
  pivot_wider(names_from = 'Run', 
              values_from = 'PG.MaxLFQ')

3.6.2 Impute Missing Values

With this wide data frame, we will now impute missing values.

We have an internally created function to perform this imputation, it is as follows:

imputed_values <- impute_normal(df = diann_peptide_filtered_wide %>%  
                                  select(-Genes) %>% 
                                  column_to_rownames(var = 'Protein.Group'),
                                width = 0.3,
                                downshift = 1.8)

Now there should be no missing values present in the data. The next thing we need to do is make this data back into a format more convenient to work with (i.e., a tibble) and add back in particular columns (e.g. Protein.Group and Genes)

imputed_data_wide_df <- imputed_values %>% 
  as.data.frame() %>% 
  rownames_to_column(var = 'Protein.Group') %>%
  bind_cols(y = diann_peptide_filtered_wide %>% 
              select(Genes)) %>% 
  relocate(Genes, .after = 'Protein.Group')

imputed_data_df <- imputed_data_wide_df %>% 
  pivot_longer(cols = 3:last_col(),
               names_to = 'Run',
               values_to = 'PG.MaxLFQ') %>% 
  left_join(diann_peptide_filtered_data %>% 
              select(-c(PG.MaxLFQ, group)),
            by = c('Run', 'Protein.Group', 'Genes')) %>% 
  left_join(diann_peptide_filtered_data %>% 
              distinct(Run, group),
            by = 'Run') %>% 
  relocate(Run, .before = 'Protein.Group') %>% 
  relocate(group, .after = 'Run')
